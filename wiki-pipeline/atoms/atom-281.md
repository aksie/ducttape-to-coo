## Claim

For freelancers (relevant under the wet DBA), make IP transfer explicit because it does not transfer by default, include an indemnity so liability for false self-employment can pass to the freelancer, and draft so the arrangement genuinely reads as freelance, not employment.

## Source quote or paraphrase

Under wet DBA: IP doesn't transfer by default — make it explicit; add an indemnity for schijnzelfstandigheid risk; draft so it reads as a genuine freelancer (doesn't solve everything).

## Why

Default rules leave freelancer-built IP outside the company, and false-self-employment enforcement creates real liability — explicit IP and indemnity clauses limit both.
