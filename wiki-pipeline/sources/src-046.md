## Summary

Internal-communication series post arguing companies need channels with explicit jobs, not more channels. Slack/Teams for coordination and movement; docs for memory; email for formal announce/summarize with link to durable source; meetings for debate that chat cannot resolve. Practitioner-framed ops writing aimed at scaling startups.

## Notable claims

- Slack is for coordination, fast questions, alerts, and social connection — weak for durable decisions and long-term memory
- Rule: use chat for movement, not memory — move decisions to a durable record
- Email announces or summarizes but should link to the authoritative doc and name where discussion happens
- Docs are the source of truth; Slack threads are where decisions get lost
- A company needs channels with jobs — where to look, decide, escalate, record, and where not to create noise
- Playbook should define what moves from chat to source of truth and which channel is the fast lane for incidents
