## Summary

Operator-oriented offboarding guide for startups and scale-ups (Jan 2026). Ten-step checklist covering access revocation, knowledge transfer, payroll, exit interviews, communications, compliance, and alumni relations. Actionable DRIs per workstream; some steps target larger teams (Okta automation, SOC 2) but core structure applies at first hires.

## Notable claims

- Master access list of every tool, categorised by sensitivity, maintained before any departure event
- Knowledge transfer should start 2–3 weeks before last day with a standardised doc template (Notion/Google Doc)
- Exit interview by People Ops or neutral third party; separate manager closure conversation for appreciation
- Every checklist item needs a named DRI — manager owns handover, IT owns assets/access, People Ops coordinates
- Equipment return uses signed checklist; company-generated shipping labels for remote returns
