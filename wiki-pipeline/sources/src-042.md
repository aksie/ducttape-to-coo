## Summary

HN thread (Nov 2024, 270+ comments) from a remote startup founder asking how to communicate better. Dominated by practitioner operators describing Slack norms, documentation habits, async culture, and in-person cadence. Strong themes: public channels over DMs, Slack is not the system of record, decisions must be codified elsewhere, and teams need explicit permission not to read everything.

## Notable claims

- Default to public Slack channels; if a DM reaches a decision, post the consensus back to a channel so it is searchable
- Slack is ephemeral — valuable content becomes undiscoverable; parallel wiki/docs processes are required
- If anything is decided in Slack, it must be codified in a wiki or handbook
- Teach people they do not need to read every message — Slack is a shared brain, not an inbox to clear
- Meme/social channel should be separate from operational source-of-truth
- Pairing/huddle solutions should be posted as text afterward, not screenshots only
- At least one annual in-person meetup materially improves remote culture; quarterly is often too heavy
- Confluence-per-team pattern: Slack questions should point to where documentation lives
