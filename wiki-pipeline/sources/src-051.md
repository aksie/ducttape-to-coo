## Summary

HN thread (2022, 200+ comments) on minimum IT security for early-stage startups. Multiple operators recommend paired onboarding/offboarding checklists alongside SSO, password manager, and CI/CD hardening. Debate on when SSO is worth the cost at pre-PMF, but consensus that you must know who has access when people leave.

## Notable claims

- Create explicit onboarding and offboarding checklists as part of baseline security — not optional at first hires
- Google Workspace (or similar) as identity provider simplifies offboarding vs per-app accounts
- Password manager (1Password, etc.) with no passwords in Slack; rotate immediately if credentials appear in chat
- SSO reduces manual per-service revocation — worth doing early even if not every app supports it
- Master inventory of services and who holds admin rights — critical when contractors or early hires rotate
