## Summary

HN thread (2024, 180+ comments) on team credential management at small companies. Strong practitioner consensus: per-user accounts over shared passwords; cloud password manager with collections; on departure remove from manager first then rotate shared creds; SSO where affordable. Warns that most companies skip rotation — a common offboarding gap.

## Notable claims

- When someone leaves: remove from password manager first, then rotate passwords they could access; team gets new creds automatically via manager
- Shared passwords are sometimes unavoidable — rotate every time anyone with access leaves
- Prefer individual accounts per service so you can disable one person without disrupting everyone
- Shared spreadsheets for passwords are too easy to mess up and audit poorly
- SSO-connected services shut down automatically when the user account is disabled
