Add the Virtual COO to your setup

Beta — please test, use with caution. The Virtual COO is in active testing. You're welcome to try it; guidance may be incomplete or wrong. All feedback is very welcome.

The Virtual COO is grounded in the Duct Tape to COO operations wiki, which it reads over HTTP — no local copy of the product repo required. Drop it into whatever AI tool you already use.

This page covers installing the skill. Setting up the recurring weekly & quarterly check-ins is separate — see Set up your check-ins.

What every setup needs
  1. The skill files — the virtual-coo/ folder (on GitHub): SKILL.md, reference.md, tone-of-voice.md, prompts/.
  2. A company-state.md in your own workspace — copy the template, fill it in, keep it in your company project (not this repo). This is the COO's memory.
  3. Web access — so it can fetch the knowledge base. No web access? Clone the repo and it falls back to local files.

Quick start

  1. Get the skill files. Clone once, copy the folder into your skills directory: 
    git clone https://github.com/aksie/ducttape-to-coo.git
cp -r ducttape-to-coo/.cursor/skills/virtual-coo ~/.cursor/skills/

    Project-scoped instead (skill travels with one repo)? Use .cursor/skills/virtual-coo/ inside that project. Full folder on GitHub: .cursor/skills/virtual-coo.

  2. Add company-state.md to your company workspace — the project where you actually run the company (not the ducttape-to-coo repo): 
    cp ducttape-to-coo/docs/company-state.md ~/path/to/your-company/company-state.md

    Or open the template on GitHub and save it as company-state.md in your project root.

  3. Open your company project in your AI tool (Cursor, Claude Code, …) with the skill installed. Say:

    "Be my virtual COO."

    First time? The skill runs a short intro and asks about your stage, headcount, revenue, and funding — then writes company-state.md for you. You don't need a filled-in file before you start.

  4. Optional — recurring check-ins. Pick a weekly time on the check-in setup page; calendar reminders include the prompt to paste.

Skill files (reference)

Everything lives under .cursor/skills/virtual-coo/ in the repo:

  • SKILL.md — main instructions (required)
  • reference.md — weekly arc, health check, event playbooks
  • tone-of-voice.md — how to sound (peer, not auditor)
  • prompts/ — copy-paste prompts for calendar reminders
  • SETUP.md — this guide as markdown (for GitHub / offline)
  • docs/company-state.md — template for your workspace (copy out)

Install per tool

Cursor SKILL.md

After cloning (see Quick start), copy the folder:

cp -r ducttape-to-coo/.cursor/skills/virtual-coo ~/.cursor/skills/   # all projects
# or, inside your company repo:
mkdir -p .cursor/skills && cp -r ../ducttape-to-coo/.cursor/skills/virtual-coo .cursor/skills/

Auto-discovered. Open your company workspace (where company-state.md lives), then ask "be my virtual COO" or any ops question. Browse the skill on GitHub: virtual-coo/.

Claude Code SKILL.md

Copy to .claude/skills/virtual-coo/ (project) or ~/.claude/skills/virtual-coo/ (personal). The folder name must match the name: in SKILL.md — it does.

Invoke with /virtual-coo, or let it trigger from the description.

OpenCode SKILL.md / AGENTS.md

OpenCode reads .claude/skills natively, so a .claude/skills/virtual-coo/ folder works out of the box.

Or wire it as a command (.opencode/command/virtual-coo.md), an agent (.opencode/agent/virtual-coo.md), or add the skill text to AGENTS.md.

Hermes Agent / OpenClaw SKILL.md

Both use the same SKILL.md format. Drop virtual-coo/ into their skills directory (Hermes: ~/.hermes/skills/virtual-coo/; OpenClaw: its skills folder or via ClawHub).

Invoke from chat — CLI or a messaging channel (Telegram, Slack, …).

Generic rules / system prompt

For Cline, Continue, Aider, or a raw API/SDK: paste SKILL.md (and reference.md when needed) into the tool's rules or system prompt.

Give it file access to your company-state.md and web access to the knowledge base.

Configure the knowledge source

Default KNOWLEDGE_BASE_URL (in SKILL.md): https://raw.githubusercontent.com/aksie/ducttape-to-coo/main

Usually you change nothing — the skill fetches the wiki over HTTP. Offline? Keep a local clone and the skill falls back to disk.

First run

"Be my virtual COO."

First conversation: the COO introduces itself (beta note), asks about your company — name, headcount, stage, revenue, funding, recent events — and writes company-state.md. Then a light read on what matters at your stage. It may point you to the check-in setup page for recurring reminders.

After that, ask ops questions any time, say "weekly check-in", or paste the prompt from your calendar.
