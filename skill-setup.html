Add the Virtual COO to your setup

Beta — please test, use with caution. Guidance may be incomplete; feedback is very welcome.

Wiki-grounded ops partner for your AI tool. You install only the skill folder — at runtime it fetches the Duct Tape to COO knowledge base (wiki pages, process data) over HTTP, so you don't need a local copy of this repo. Your AI tool needs web access; offline, a local clone works as a fallback.

Quick start

  1. Clone and copy the skill 
    git clone https://github.com/aksie/ducttape-to-coo.git
cp -r ducttape-to-coo/.cursor/skills/virtual-coo ~/.cursor/skills/

    Project-scoped? Use .cursor/skills/virtual-coo/ inside your company repo instead. Browse on GitHub →

  2. Open a project

    "Be my virtual COO."

    Any folder or repo you work in day to day. First time: intro + questions about your company — then the COO creates company-state.md for you. When you're ready for a weekly rhythm, the COO can point you to calendar reminders — first conversation or later.

Install per tool

Same skill folder everywhere — only the destination path changes.

Cursor

Copy the virtual-coo/ folder to ~/.cursor/skills/virtual-coo/ to use it in every project, or to .cursor/skills/virtual-coo/ inside one repo. Cursor discovers it automatically — open a project and ask "be my virtual COO".

Claude Code

Copy the folder to .claude/skills/virtual-coo/ in your project, or to ~/.claude/skills/virtual-coo/ for all projects — the folder name must match the name: field in SKILL.md. Invoke with /virtual-coo, or let Claude pick it up from the skill description.

OpenCode

Copy the folder to .claude/skills/virtual-coo/ — OpenCode reads that path natively. Or wire the skill as a command, an agent, or paste it into AGENTS.md.

Hermes Agent / OpenClaw

Drop the virtual-coo/ folder into their skills directory (e.g. ~/.hermes/skills/virtual-coo/). Both tools use the same SKILL.md format — invoke from chat on CLI or a channel like Telegram or Slack.

Other (Cline, Continue, Aider, API)

Paste SKILL.md (+ reference.md when needed) into rules or system prompt. Give file access to company-state.md and web access to the wiki.
