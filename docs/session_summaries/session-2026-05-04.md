## Pipeline architecture

```
Sources → Atoms → Candidate Claims → Draft + Trail → Approval → Published Wiki
(Phase 0)   (Phase 1)       (Phase 1½)          (Phase 2)    (Phase 3)    (Phase 4)
```

### File types and locations

| File | Location | Purpose |
|------|----------|---------|
| Source | `wiki-pipeline/sources/src-NNN.md` | Raw source material with metadata |
| Atom | `wiki-pipeline/atoms/atom-NNN.md` | Single extracted claim from a source |
| Candidate Claims | `wiki-pipeline/entries/{p}/{ph}/candidate-claims.md` | Parking lot for claims before synthesis |
| Draft | `wiki-pipeline/entries/{p}/{ph}/draft.md` | Synthesized wiki entry with `<!-- claim-id: c-NNN -->` markers |
| Trail | `wiki-pipeline/entries/{p}/{ph}/trail.md` | Audit trail: each claim → supporting atoms |
| Approval | `wiki-pipeline/entries/{p}/{ph}/approval.md` | Review decisions per claim |
| Published | `wiki/processes/{category}/{process}--{phase}.md` | Final published page |

### Server endpoints (localhost:8765)
| Endpoint | Method | Description |
|----------|--------|-------------|
| `/` | GET | Serve `approval-tool.html` |
| `/api/entries` | GET | List all entries with approval summary |
| `/api/entry/{process}/{phase}` | GET | Full entry data (draft, trail, approval, atoms, sources) |
| `/api/approve` | POST | Save a claim decision to `approval.md` |

## Key decisions and conventions

### Naming conventions
- **Sources**: `src-NNN.md` (zero-padded, e.g. `src-020-gitlab.md`)
- **Atoms**: `atom-NNN.md` (zero-padded, e.g. `atom-067-gitlab.md`)
- **Claims**: `c-001` (per-draft, restarts for each process/phase cell)
- **Process slugs**: `strategic-ops`, `financial-ops`, `people-ops`, `legal-ops`, `revenue-ops`
- **Phase slugs**: `foundation`, `first-hires`, `early-scale`, `growth`, `scaled`
- **Published pages**: `{process}--{phase}.md` (e.g. `1.1--foundation.md`)

### Wiki page structure (5 sections)
1. `## What good looks like at this phase` — from `target_state` atoms
2. `## What you actually need to do` — from `action` atoms
3. `## Warning signs you're behind` — from `warning_sign` atoms, grouped by:
   - `output_quality` — process output is wrong or risky
   - `founder_time` — bottleneck person spending time on wrong things
   - `process_entry` — people don't know how to participate
4. `## How this evolves next` — from `evolution` atoms
5. `## Tools & resources` — from `tool_resource` atoms

### Atom conventions
- **`extracted_by`**: `human:username` (elevated weight) or `llm:model-slug` (standard)
- **Max 7 bullets per section** in synthesized drafts (combined across warning sign categories)
- Each bullet: **bold claim** + indented 2–4 line explanation + `<!-- sources: src-NNN -->`
- Each claim marked with `<!-- claim-id: c-NNN -->` starting from `c-001`

### Approval flags
- `too_generic`, `conditional`, `vendor_biased`, `geographically_biased`, `needs_practitioner_check`, `missing_why`
- **Reviewer prefixes**: `human-insight:` (append sentence), `human-rewrite:` (rewrite with practitioner knowledge)
